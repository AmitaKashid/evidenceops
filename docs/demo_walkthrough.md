# Five-minute demo walkthrough

## Setup

Start the backend and frontend, then open `http://localhost:3000`.

## 0:00 to 0:45 — the problem

Open **Control room** and explain the premise:

> A power user should be able to delegate a multi-step analysis task, but every important conclusion needs evidence, calculation checks, monitoring, and an explicit approval step.

## 0:45 to 2:00 — task execution

Open **Task studio**. Submit the default vendor-selection task. Show:

- the task contract created from the broad user request
- retrieval across requirements, vendor security evidence, policy rubric, and pricing schedule
- deterministic three-year TCO calculation
- structured decision brief with claims and sources

## 2:00 to 3:15 — reliability layer

Open the verifier panel and demonstrate:

- citation coverage and precision
- numeric reconciliation
- unresolved questions and escalation requirement
- proof that the system does not auto-approve a consequential recommendation

Use the evidence ledger details to show how a reviewer can inspect source excerpts.

## 3:15 to 4:00 — human approval

Record a reviewer decision. Explain that this is deliberately a read-only workflow: the agent prepares analysis but does not send messages, execute procurement, or modify external systems.

## 4:00 to 5:00 — continuous improvement

Open **Evaluation lab** and run the TaskBench. Explain:

- all profiles see the same corpus, task cases, tools, output schema, and verifier
- quality, evidence grounding, numeric accuracy, escalation behavior, cost, and latency are visible together
- real failures become new regression cases before a workflow change is released

## Close

> The differentiator is not “many agents.” It is a governed execution system that makes task quality measurable and continuously improvable.
